NSM Footnotes requires you to wrap {exp:nsm_footnotes} around the content you want processed. Checkout the sample templates before reading the following reference.

{exp:nsm_footnotes}

{exp:nsm_footnotes
([
    string ref_prefix 
    string fn_prefix 
    string ref_class 
    string fn_class
    string entry_id 
])}

Parameters

ref_prefix [optional]

ref_prefix="ref-"

Prefixed to the id="" attribute of the reference anchors generated in the template. The numerical index of the reference will be concatenated with the ref_prefix param. Example output where ref_prefix="ref-":

<a href="#fn-0" id="ref-0" class="ref">0</a>

fn_prefix [optional]

fn_prefix="fn-"

Similar to ref_prefix this parameter is prefixed to the id="" attribute of the footnote anchors generated in the footnote index. The numerical index of the footnote will be concatenated with the fn_prefix param. Example output where fn_prefix="fn-":

<a href="#ref-0" id="fn-0" class="fn">0</a>

ref_class [optional]

ref_class="ref"

The class="" attribute applied to the reference anchor. Example output:

<a href="#fn-0" id="ref-0" class="ref">0</a>

ref_class [optional]

fn_class="fn"

The class="" attribute applied to the footnote list items. Example output:

<a href="#fn-0" id="ref-0" class="fn">0</a>

entry_id [optional]

entry_id="1"

The entry id is used to create a unique tag pair. This addresses templates with multiple {exp:nsm_footnotes} tags.

Single variables

{footnotes}

{footnotes}

This tag can be used a single tag or as a tag-pair but cannot appear as different formats in a single {nsm:footnotes} tag-pair. Using {footnotes} as a single tag will generate the output for you in a pre-determined format.

This tag loops over all of the unique references and outputs links back into the document and the footnote content.

{footnote_total_results}

The total number of footnotes to display.

{footnote_refs_total_results}

The total number of references to display. When using named references a footnote may have more than one reference in the content.

Variable pairs

{footnotes}…{/footnotes}

<ul>{footnotes}
    <li>
        {footnote_refs}
            <a href="#ref-{footnote_ref_id}" id="fn-{footnote_ref_id}">
                {footnote_ref_id}
            </a>
        {/footnote_refs}
        {footnote_content}
    </li>
{/footnotes}</ul>

This tag can be used a single tag or as a tag-pair but cannot appear as different formats in a single {nsm:footnotes} tag-pair. Using {footnotes} as a single tag will generate the output for you in a pre-determined format.

Loop over all of the unique references and outputs links back into the document and the footnote content. This tag also has a number of nested tags for outputting a single footnotes references and footnote content.

{footnote_content}

This tag pair must be nested inside of the {footnotes}…{/footnotes} tag pair above. The footnote content. This is the content that was found inside of double-bracket-pairs ([[…]]) in the {exp:nsm_footnotes} tag-pair.

{footnote_refs}…{/footnote_refs}

This tag pair must be nested inside of the {footnotes}…{/footnotes} tag pair above. Loops over the references for a single footnote. The following nested tags are available:

{footnote_ref_id}
This tag pair must be nested inside of the {footnote_refs}…{/footnote_refs} tag pair above. The footnote reference id. This is a unique id assigned to every reference found in the content. It is used to link the footnote anchors to the anchors generated in the content.
